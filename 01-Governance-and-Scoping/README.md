# Phase 1: Governance, Organizational Context & Scoping

## 📋 Overview
This directory houses the foundational governance documentation for NexaLearn Inc. In alignment with standard risk frameworks such as **ISO/IEC 27001 (Clause 4)** and **NIST CSF 2.0 (Organizational Profile)**, an effective information security program must begin by establishing clear business context, identifying external regulatory requirements, and defining strict system scoping boundaries.

Without a well-defined scope, security programs suffer from "scope creep" or fail to protect critical data repositories. This document sets the "rules of engagement" for all subsequent risk management activities.

---

## 🔍 Core Objectives
* **Business Profile Mapping:** Defining NexaLearn's operational footprint as a high-availability, multi-tenant cloud-native SaaS platform serving 520,000+ global users.
* **Regulatory Compliance Boundaries:** Identifying critical legal and data privacy compliance touchpoints, specifically **FERPA** (for US student record privacy) and **GDPR** (for European user protection).
* **System Boundary Isolation:** Defining exactly what infrastructure, data, and services are **In-Scope** (e.g., AWS production environments, PII database clusters) versus **Out-of-Scope** (e.g., corporate HR, employee home offices) to streamline the auditing lifecycle.
