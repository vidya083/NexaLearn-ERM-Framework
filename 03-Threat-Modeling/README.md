# Phase 3: Threat Modeling & Threat/Vulnerability Pairs

## 📋 Overview
This directory contains the threat modeling documentation for the NexaLearn SaaS platform. In accordance with enterprise risk management standards, security cannot begin with controls; it must begin with a clear understanding of active adversarial behaviors and inherent system design weaknesses.

This phase bridges the gap between our **Phase 2 Asset Inventory** and our upcoming risk treatments by identifying and analyzing specific **Threat/Vulnerability Pairs** that pose an existential risk to the business.

---

## 🔍 Core Objectives
* **Adversarial Mapping:** Identifying external cybercriminals, insider threats, and systemic threat vectors relevant to a cloud-native EdTech architecture.
* **Vulnerability Analysis:** Cataloging process and technical weaknesses (such as legacy authentication, unpatched systems, and overly permissive API configurations).
* **Exploit Vector Modeling:** Analyzing active, real-world breach methodologies—specifically simulating vectors inspired by the modern **Canvas LMS credential incidents** and lateral **WannaCry-style ransomware** outbreaks.
